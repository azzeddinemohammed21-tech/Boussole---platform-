# بوصلة+ Platform — المرحلة ١

هذا هو التسليم الأول من منصة بوصلة+، مبني بـ Next.js 14 + TypeScript + Tailwind CSS + Supabase.

## ما تم إنجازه في هذه المرحلة

- هيكلة مشروع Next.js كاملة (App Router).
- الصفحة الرئيسية بتصميم عربي RTL خاص بهوية بوصلة+ (كُحلي + نحاسي، إبرة بوصلة كعنصر بصري).
- صفحتا تسجيل الدخول وإنشاء الحساب، متصلتان فعليًا بـ Supabase Auth.
- لوحة مستخدم أولية (محمية، تتطلب تسجيل دخول).
- مخطط قاعدة بيانات ابتدائي (`supabase/schema.sql`) مع سياسات أمان (RLS).
- SEO أساسي (عنوان، وصف، Open Graph) بالعربية.

## التشغيل محليًا

```bash
npm install
cp .env.example .env.local   # ثم ضع مفاتيح Supabase الخاصة بك
npm run dev
```

افتح http://localhost:3000

## ربط Supabase (5 دقائق)

1. أنشئ مشروعًا مجانيًا على supabase.com.
2. من SQL Editor، شغّل محتوى `supabase/schema.sql`.
3. من Project Settings > API، انسخ `Project URL` و `anon public key` إلى `.env.local`.

## النشر المجاني

- **Vercel**: اربط المستودع من GitHub، وأضف نفس متغيرات `.env.local` في إعدادات المشروع على Vercel. النشر تلقائي بعد كل push.
- **Cloudflare Pages**: بديل مجاني إذا فضّلت عدم استخدام Vercel (يتطلب إعداد `@cloudflare/next-on-pages`).

## خارطة الطريق — المراحل القادمة

كل مرحلة تُسلَّم كملفات جاهزة تُضاف مباشرة فوق هذا الأساس:

| # | المرحلة | المحتوى |
|---|---------|---------|
| 2 | نظام اختبار اكتشاف الكفاءات | الأسئلة، منطق التصحيح، صفحة النتائج المرتبطة بمنهجية بوصلة+ |
| 3 | لوحة الإدارة | إدارة المستخدمين، الاختبارات، الدورات، الفرص |
| 4 | محرك المطابقة الذكي (v1) | مطابقة نتائج الاختبار بالدورات والفرص المناسبة |

| 8 | النسخة الإنجليزية + i18n كامل | تبديل لغة، مسارات `/en` |

## ملاحظة مهمة

هذا تسليم برمجي حقيقي يعمل، وليس نموذجًا شكليًا. الأقسام غير المبنية بعد (الاختبارات، لوحة الإدارة، محرك المطابقة) مذكورة بوضوح في خارطة الطريق أعلاه، ولم يُدّعَ أنها موجودة.

أخبرني بأي مرحلة تريد البدء بها تاليًا، وسأبنيها فوق هذا الأساس مباشرة.
